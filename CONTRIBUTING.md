# Contributing to Zombie API Discovery and Defence Platform

## Branching
- Use feature/ or bugfix/ prefixes (e.g., feature/api-scanner)
- Keep main branch stable

## Commits
- Write clear, descriptive commit messages
- Commit often, push regularly

## Pull Requests
- Open PRs from feature branches
- Assign reviewers
- Address feedback before merging

## Code Style
- Follow best practices for Python/Node
- Document major functions and modules

## Issues
- Use GitHub Issues for bugs, features, and questions

## Communication
- Keep discussions in PRs or Issues for transparency
